# RegEx (Expressões Regulares)

Úteis para realizar pesquisas ou substituir conteúdos.  
São muito utilizadas no terminal Linux para buscar, filtrar e manipular textos e arquivos.  

"canivete suíço" da computação


    .*    É o "coringa" mais poderoso das Expressões Regulares.
    \d    Corresponde a qualquer dígito numérico (equivalente a [0-9]).
    \d+   Encontra uma ou mais ocorrências consecutivas de dígitos numéricos (0-9).

###### Utilização

**Validação de Formulários**:  Garante que o dado siga um padrão (molde) antes de ser salvo.

É onde o Regex roda bilhões de vezes por segundo. Cada vez que alguém no mundo digita um e-mail, senha ou CPF, um Regex é disparado. 

👉️ Uso mais comum do regex é na **Validação de Formulários**, o uso mais "pop" e padronizado.

**Terminal Bash**: Filtrar e buscar textos, automação.
**Editores de código (Nano, Gedit)**: O Regex serve para o famoso Localizar e Substituir.

👉️ Regex também é muito utilizado no **Bash** e nos **Editores de Código**, além dos formulários onde são o "Campeão de Audiência".

**Programação**: (JavaScript, Bash Scripting, Python...)

Quase toda linguagem de programação tem suporte nativo a Regex. Os programadores usam para:
Limpeza de Dados (Data Scraping): Pegar um texto gigante de um site e extrair apenas os preços ou números de telefone.
Refatoração: Mudar o nome de uma função em centenas de arquivos de uma vez só, mas apenas se ela estiver dentro de um contexto específico.


**Servidores Web**: (Redirecionamento) (Servidores como Apache ou Nginx usam Regex para ler a URL que você digitou e "traduzir" para a nova.)

**Segurança e Logs (Cybersegurança)**:

**Tradução e Revisão de Texto**:
Ferramentas profissionais de tradução (CAT Tools) usam Regex para garantir que termos técnicos não sejam traduzidos ou para padronizar datas e unidades de medida de um idioma para outro (ex: trocar . por , em números decimais).

###### Onde o Regex NÃO deve ser usado?

Existe uma brincadeira clássica na computação: "Você tem um problema e decide usar Regex. Agora você tem dois problemas."

O Regex é péssimo para entender contexto. Por exemplo, usar Regex para ler HTML complexo é considerado uma má prática, porque o HTML tem "ninhos" (tags dentro de tags) que podem confundir o padrão do Regex. Para isso, existem ferramentas chamadas **Parsers**.
