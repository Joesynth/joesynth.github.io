## HTML, CSS e JS


<img src="https://gitlab.com/uploads/-/system/project/avatar/62233751/HTML.png" width="120" />**HTML, CSS e JS**: O trio de linguagens mais comuns da internet, e que até trabalham juntas. A linguagem HTML, por exemplo, é tão básica que por si só já é capaz de construir um site. Começar com HTML é um bom caminho. Primeiro porque é uma linguagem simples, segundo porque dá a base para as outras linguagens (principalmente a CSS), e terceiro porque vamos interagir com outras ferramentas do computador, como é o caso de Editores de Códigos.  

Vide o *link* [HTML, CSS, JS](html-css-e-js).  

## JSON

<img src="https://upload.wikimedia.org/wikipedia/commons/thumb/c/c9/JSON_vector_logo.svg/3840px-JSON_vector_logo.svg.png" width="120" />**JSON**: Parece nome de linguagem também, mas neste caso não é. No JSON podemos organizar dados simplesmente pela forma de como escrevemos; inclusive o JSON é um formato de texto e que pode ser salvo no computador (usa extensão .json).  


**Lista com dois times**: Neste exemplo temos dados sobre dois times.

    [
      {
        "time": "Flamengo",
        "estado": "RJ"
      },
      {
        "time": "Santos",
        "estado": "SP"
      }
    ]

Diferente de uma linguagem de computador, que tem símbolos e palavras estranhas que só a máquina ou um [programador](/profissionais-de-informatica) entende, o JSON é compreensível por qualquer pessoa _e a máquina também! :). Outra coisa bacana é que o JSON é filho da linguagem JavaScript. Então se você se habitua a lidar com o formato JSON, certamente isso abre caminho para aprender a JavaScript com mais facilidade.

Leia mais sobre [JSON](/json).



## Bash

<img src="https://upload.wikimedia.org/wikipedia/commons/thumb/4/4b/Bash_Logo_Colored.svg/960px-Bash_Logo_Colored.svg.png" width="120" />**Bash**: No computador. ele é praticamente "sinônimo" da *telinha preta* (o Terminal). Em vez de operar o computador pela interface gráfica (com o mouse) usamos o teclado e a tela preta.
Desde coisas básicas, como criar ou apagar arquivos, até coisas complexas podem ser feitas pelo Bash. 
Conhecê-lo é vital para se trabalhar com TI. A própria web "está de pé" no mundo inteiro com profissionais usando o Bash a todo momento. Cerca de 95% da estrutura da internet é Bash. Ou seja, ele é o coração.




## Gedit e Nano

<img src="https://upload.wikimedia.org/wikipedia/commons/thumb/c/ca/Gedit-logo-clean.svg/1280px-Gedit-logo-clean.svg.png" width="60" /> <img src="https://upload.wikimedia.org/wikipedia/commons/thumb/8/8a/Gnu-nano.svg/250px-Gnu-nano.svg.png" width="60" />**Gedit eNano**: Ambos são editores de códigos e presentes no Linux. Enquanto no Gedit utilizamos a interface gráfica comum (com *mouse* etc), no Nano usamos a "telinha preta". O Nano é acessado pelo Bash, daí já podemos imaginar o quanto ele é comum.

👉️ Neste momento em que escrevo estou usando o Gedit, mas poderia ser o Nano. 




## Markdown

<img src="https://upload.wikimedia.org/wikipedia/commons/4/48/Markdown-mark.svg" width="120" />**Markdown**: É uma daquelas coisas maravilhosas que a informática já criou. Markdown é uma linguagem para escrever textos e num passe de mágica criarmos títulos grandes e bonitos, colocarmos uma palavra em negrito ou mesmo inserirmos uma imagem no texto. É uma linguagem simples, com uns 10 "comandinhos".

🤔️ Markdown deveria ser ensinado nas escolas logo desde que surgiu, lá pelos anos 2000.  
👉️ Óbvio que neste momento escrevo estas linhas usando exatamente esta linguagem, além do editor Gedit.



## RegEx

<img src="https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcQieMsMZKAZP8LY7GPahe_Kj9RL9fzAAud5qA&s" width="120" />**RegEx** (Expressões Regulares): É uma "mini-liguagem", digamos uma linguagem que descreve padrões de caracteres. Sabe quando alguém digita um e-mail ou CPF num campo de formulário? Pois é, o RegEx vai lidar com isso. Num editor de código, como o Nano ou o Gedit que falamos aqui, o recurso de RegEx nos permite substituir conteúdos, enquanto no Bash podemos usar para buscar, filtrar e manipular textos. Portanto o RegEx é útil em diversas situações.

👉️ Uma forma simples de começar a lidar com RegEx é pelo seu editor de códigos, substituindo palavras.




## Console Firefox

<img src="https://upload.wikimedia.org/wikipedia/commons/thumb/a/a0/Firefox_logo%2C_2019.svg/1280px-Firefox_logo%2C_2019.svg.png" width="120" />**Console Firefox**: Semelhante ao Bash, porém em vez de controlar o sistema operacional ele permite controlarmos o que vemos numa página web. É possível até escrevermos código, como se fosse um editor Nano ou Gedit. Nele iremos lidar com linguagens HTML, CSS e JavaScript.



## 🎉️ Bônus!

<img src="https://images.icon-icons.com/2699/PNG/512/gitlab_logo_icon_169112.png" width="120" />🇳🇱 **GitLab**: É uma rede social de programadores. Em vez de publicarmos fotos, como no Instagram, no GitLab postamos códigos de programas que estamos criando.  

Ter uma conta no GitLab é uma ótima experiência para quem está iniciando no mundo da programação. Aliás, mega corporações também o utiliza _embora de forma velada para que ninguém veja seus códigos (caso da IBM, Siemens e NVIDIA).

Um jovem programador encontrar um desenvolvedor de softwares famoso no GitLab é como um desenhista iniciante achar o perfil do ilustrador [Pawel Kuczynski](/pawel) no Instagram.

👉️ O ponto forte do Gitlab não é "seguir" usuários ("famosos") e dar "estrelinhas", mas ser uma ferramenta de trabalho colaborativo, com programadores de qualquer lugar do mundo podendo ajudar num projeto (de software), como o GIMP ou o VLC.
