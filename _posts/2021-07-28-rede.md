# Rede

É um conjunto de dispositivos ligados que podem trocar informações entre si além de compartilhar recursos e aplicativos.



## Introdução

###### O que pode ser ligado?

Pode ter ligado...




Computadores PCs, tablets, smartphones, impressora, servidores de arquivo, câmeras de vigilância...

###### Ela compartilha...

Arquivos, dados. Ex: computador que tem um banco de dados de clientes e acessado por vários outros computadores (estes consultando dados como clientes, produtos, etc...); também aplicativo que dá acesso no servidor pra colocar pedido de compra. Também compartilha recursos como impressão. A impressora será acessada pelos PCs ou um computador na rede é o servidor de impressão.

###### Nó Central, Nó de Controle e Nós de Extremidade

📚 *Central Node, Control Node, Endpoints*

Abaixo fazemos uma analogia com uma estrada.

|Nó Central | Nó de Controle | Nó de Extremidade |
| :--- | :--- | :--- |
|switch, hub ou roteador | **servidor**, mainframe ou um PC (master) | desktop, **celular**, notebook, dispositivo IoT |
|Equivale ao entroncamento das estradas | Equivale à central de monitoramento de tráfego | Equivale aos destinos finais (casas, garagens) |
|Camada Física / Infraestrutura | Camada Lógica / Aplicação | Camada de Interface / Usuário Final |
|Administrador de Rede|Profissional de DB e TI|Pessoas em geral (Usuários)|
|Interface: Bash |Interface: Bash e GUI|Interface: GUI|
|Username + SSH Key / Senha|Username + SSH Key / Senha|E-mail / Senha|


O **nó Central** e o **nó de Controle** equivalem a parte do **servidor** enquanto os **nós de extremidade** a parte do **cliente**.  

No Modelo Cliente/Servidor, os **nós de extremidade** (*endpoints*) são os **clientes**.  
**Pontos de extremidade**: Eles são chamados assim, justamente, porque são o início e o fim da interação. O usuário toca na tela (cliente), o pedido viaja pela rede (Nó Central) até o servidor (Nó de Controle), que processa e devolve a resposta para o aparelho (cliente).  

**SSH Key** (Chave SSH): um método de autenticação muito mais seguro que o login por senha/e-mail (do Endpoint). Para o usuário comum, o foco é a conveniência. Para o Administrador e o Profissional de TI, o foco é a impenetralidade.  
**Leaf Node** termo semelhante a Endpoints.  





###### Administrador (Admin/SysAdmin)

Login/Username: é o que a pessoa usa para entrar no Uber, por exemplo.
UID (User Identifier): é o "código de máquina" do motorista. É a Chave Primária no banco de dados da Uber

🤔 Curiosidade sobre o "Admin": o Admin/SysAdmin surge na coluna da Pessoa. No entanto, tecnicamente, o Admin muitas vezes nem usa um navegador comum (Chrome/FF). Ele costuma usar um Agente do Usuário chamado SSH ou Terminal para dar comandos diretos ao Nó (Servidor), sem interface visual.

###### Modelos de Controle

A ideia de "controle" ou "propriedade" depende da arquitetura utilizada:

|Redes Cliente-Servidor|Redes Ponto-a-Ponto (P2P)|
|:---:|:---:|

## 👤️ Usuário

|Usuário com privilégios elevados|Usuário Comum|
|:---|:---|
|Gerencia a infraestrutura da rede ou do servidor|Quem consome os dados (no celular ou PC)|


O usuário é quem acessa um sistema, é o 'cliente' no contexto cliente/servidor. Na rede de computadores, nos servidores é gerenciado arquivos bem como os próprios usuários. Gerenciar usuários é uma das tarefas comuns na administração de sistemas.

**Grupos**: um grupo (ou mais) pode ser criado pelo (profissional) administrador de redes quando há muitos usuários ou alguma segmentação é necessária (departamentos, divisões etc.); isso ajuda na organização. Por exemplo, podemos ter no sistema duma empresa os grupos "diretores" e"operadores".

Ou seja, é perfeitamente possível e é exatamente como grandes sistemas (como Uber ou WhatsApp) funcionam. No servidor (Nó de Controle), não se enxerga os usuários como indivíduos isolados, mas sim como objetos dentro de grupos ou listas.

No servidor, pode-se criar uma lógica onde o "Grupo A" (50 pessoas, por ex) recebe um tipo de propaganda (por e.) e o "Grupo B" (200 pessoas) recebe outra.


**Permissões**: grupos podem ter permissões diferentes, e definir essas permissões é uma etapa importante no gerenciamento. Nas permissões se diz o que cada grupo está autorizado a fazer. No exemplo dado acima, o grupo "diretores" é dado o poder de editar arquivos, enquanto o time de "operadores" somente poderá ler (mas sem poder editar).

Portanto, um grupo serve para segmentar usuários bem como dizer o que estas pessoas estarão autorizadas a fazer. Pode-se definir uma hierarquia adicionando regras, e assim é possível que um grupo tenha muita autonomia enquanto outro bem menos. Para mais sobre esse tema veja <a href="https://joesynth.wordpress.com/2020/05/11/bash-shell/">Bash Shell</a>.

⛲ blog.wieczorek.com.br/index.php/2022/06/22/linux-comandos-basicos-para-gerenciamento-de-usuarios

###### VPN

(*Virtual Private Network* - Rede Privada Virtual) Rede de comunicações privada construída sobre uma rede de comunicações pública (ex: a Internet). O tráfego de dados é levado pela rede pública utilizando protocolos padrões, não necessariamente seguros. VPNs seguras usam protocolos de criptografia por tunelamento que fornecem a confidencialidade, autenticação e integridade necessáriclienteas para garantir a privacidade das comunicações requeridas. ● Ex. de protocolos normalmente aplicados numa VPN: L2TP, L2F, PPTP e o IPSec. Quando adequadamente implementados, estes protocolos podem assegurar comunicações seguras através de redes inseguras.
