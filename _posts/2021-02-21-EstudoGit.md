## Comandos

O código-fonte vai se encontrar em um dos três estados.

- git init: cria repositório, pasta "".git".
- git status: saber o *status*, dizendo a situação de trabalho dum repositório _coisas como *commits*, etc. Entretanto, se abrirmos nosso editor, formos ao diretório (do repositório) e criarmos dois arquivos (por exemplo um MD e um HTML); e, dermos novamente *git status*, o Git nos dirá que há os tais arquivos novos e que devemos acampanhá-los, "*trackeá-los*" (*track*).
- git add: começa a acompanhar arquivos pelo controle de versão. O *add* é dado logo após adicionarmos um arquivo. O *git add.* adiciona tudo enquanto *git add *.txt* adiciona todos somente do formato txt _a mesma ideia vale ora outros formatos. Feito isso, ao dar *git status* haverá referências ao *commit*: *"Changes to be commited"*.
O *git add*, em suma, passa o arquivo para a 'sala de espera", *staging area*.
- git commit: salvar as mudanças no seu projeto; é "comitar", salvar as alterações que a gente fez.
Agora, ao dar *git status* surge *nothing to commit*.
