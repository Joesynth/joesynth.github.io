<h6>Comparando com o corpo humano...</h6>
<img class="alignleft" src="https://differbetween.com/storage/img/images_6/difference_between_html_and_css_and_javascript.png" width="310" height="336" />A HTML aquivale ao esqueleto, os ossos com sua estrutura central.
A JavaScript equivale ao cérebro com toda sua rede de neurônios, fazendo com que haja movimentos. Assim como os nervos fazem os ossos se mexerem, a linguagem <em>JavaScript</em> pode fazer um objeto (como um ícone ou botão na tela) se mover. Também podemos disparar algo quando clicamos num botão (como uma caixa de alerta), fazer uma música tocar, um relógio funcionar (contando segundos e mais segundos), etc.
Já a CSS é como cuidarmos da nossa aparência: estética! O colorido das roupas, a forma do cabelo... Então, evidentemente, com o CSS podemos pintar um botão, deixar ele quadrado ou redondo e fazer muito, muito mais coisas.

<h6>Web Standards</h6>
Entende-se como <em>Normas para Web </em> e visa uma padronização e a criação de uma <em>Web Universal</em>. Estas recomendações ou normas são produzidas pela <a href="https://joesynth.wordpress.com/2021/10/31/internet-tem-dono/">W3C</a>.
<h6>O validador</h6>
Soft <em>on-line</em> da W3C de apoio ao desenvolvimento, o qual analisa, detecta e sugere correções de eventuais erros de marcação. O ideal é que durante toda etapa de desenvolvimento da marcação XHTML, esteja sendo submetido ao validador, e corrigindo se houver erros. Crie o hábito de validar com frequência.

🔗️validator.w3.org/

<strong>WHATWG</strong>

<em>Web Hypertext Application Technology Working Group</em>.

Fundada por pessoas da Apple, Mozilla e soft Opera. Esta relacionada a desenvolvimento de padrões da Web. Em suma Apple, Google, Microsoft, Mozilla.

🔗️ whatwg.org.

<h6>HTML - XML - XHTML: as diferenças</h6>
<strong>XML</strong> Linguagem de marcação e que foi a base para o XHTML.

<strong>XHTML</strong> Não admite erros na marcação. Assim, o documento deve ser bem formado e aninhado.
<strong>Aninhar</strong> convenientemente. Observe abaixo.
<pre>&lt;div&gt; &lt;p&gt; &lt;em&gt; texto enfatizado &lt;/em&gt; &lt;/p&gt; &lt;/div&gt;</pre>
<strong>Caixa da fonte</strong> XML e XHTML são sensíveis ao tamanho da caixa da fonte, inclusive atributo. Use letra minúscula.
<strong>Valor do atributo</strong> É considerado string e deve estar entre aspas, mesmo sendo número.
<strong>Atributo</strong> deve estar escrito por completo Ou seja, nome e valor.
<strong>Tag</strong> fechamento obrigatória.
<strong>Elemento vazio</strong> deve ser fechado.
